# Интеграционные e2e-тесты (Playwright)

End-to-end проверки приложения **BookCalls** в реальном браузере (Chromium). Тесты поднимают
настоящие backend и frontend и прогоняют пользовательские сценарии, убеждаясь, что фронтенд и
бэкенд работают вместе.

## Что поднимается

Playwright через `webServer` (см. `playwright.config.ts`) сам запускает:

- **backend** — `dotnet run --project ../backend/BookCalls.Api` → `http://localhost:5002`
  (in-memory store, сид-данные создаются при старте);
- **frontend** — Vite dev-сервер `http://localhost:5400`, проксирующий API-запросы на backend
  (`VITE_API_PROXY_TARGET=http://localhost:5002`). Порт намеренно нестандартный (не `5173`), чтобы
  не конфликтовать с другими локально запущенными dev-серверами.

Локально серверы переиспользуются, если уже запущены; в CI (`CI=true`) всегда поднимаются заново.

> Примечание: Vite проксирует путь `/admin` на backend, поэтому в админку тесты переходят
> клиентской навигацией (клик по ссылкам «Админ» → «Предстоящие встречи»), а не прямым `goto`.

## Покрываемые сценарии (`tests/booking.spec.ts`)

1. **Основной сценарий бронирования.** Главная → «Выбрать время» → выбор первого свободного
   слота → имя + email → «Записаться» → экран «Бронирование создано».
2. **Конфликт двойного бронирования.** Слот занимается через API «другим гостем», после чего
   попытка забронировать его же из UI приводит к ошибке 409 «Это время уже занято. Выберите
   другой слот.».
3. **Бронь в админке.** После успешного бронирования через UI бронь появляется в списке
   предстоящих встреч на `/admin/bookings` (имя и email гостя).

## Запуск локально

```bash
# из каталога e2e/
npm install
npx playwright install chromium   # первый раз — скачать браузер
npm test                          # прогнать все тесты

npm run test:headed               # с видимым браузером
npm run test:ui                   # интерактивный UI-режим
npm run report                    # открыть HTML-отчёт последнего прогона
```

Требования: установленный **.NET 10 SDK** (для backend) и **Node.js 20+**, в `../frontend`
выполнен `npm install`.

## CI

Workflow `.github/workflows/e2e.yml` устанавливает .NET и Node, ставит зависимости
`frontend/` и `e2e/`, браузер Chromium, запускает `npm test` и публикует HTML-отчёт как артефакт.
